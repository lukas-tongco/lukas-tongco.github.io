Blocked On A Simple Problem

I had a problem where I was trying to filter out certain words from a sentence but it would either keep the words I didn't want or didn't filer out everything I wanted it to.

The techniques I used to solve this are re-reading each line carefully, making sure I knew what each method is supposed to do, and debugging constantly using `console.log()`.

I felt frustrated at first because it seemed so simple but I couldn't figure it out right away. I was also determined to solve this problem because it felt like the solution was right within reach.

I learned that it's good to take breaks and move your eyes away from your code, as it allows you to "reset" and see things you easily missed. In this case, the problem was just a missing syntax that I didn't see at first because I was too focused.

Elegantly Solved A Problem

I had a problem where I was given a sentence stored in a string and I needed to capitalize the first letter of each word in it. At the time I only knew how to capitalize all characters of a single word, so I was a bit stumped at first.

The techniques I used to solve this are research, experimentation, and debugging. From online resources like W3Schools I found that there are built-in methods that help with my problem so I implemented them in my code in ways that made sense to me. After trial and error I managed to solve my problem.

I felt very satisfied after figuring out how to solve my problem without blatantly copying different code. I was also proud because I was able to translate my thought process / pseudocode into actual code.

I learned that there are a lot of built-in methods that can really save your time, so before making my own function I should do a little bit of research first to see if there are any methods available that let me do what I want to do.

Problem-Solving Techniques/Processes

  • I'm confident with pseudocode as it lets me lessen the load on my brain and I can visually see what I'm trying to achieve.
  • I'm confident with "trying something" as I naturally run code a lot just to see what happens.
  • I'm okay with the rubber ducky method, it's not something I keep in mind and is more of a last resort.
  • I'm okay with reading error messages, it can be a hit or miss for me as sometimes I understand what it's saying and sometimes I don't. It's mostly useful for figuring out what line the error is occurring on.
  • I'm confident with console.logging, I use it mostly for testing a function or specific parts inside a function. It's also helpful for making sure I'm passing the right parameters.
  • I'm confident with Googling, I use it a lot to figure out what certain things do but I always make sure I understand what I'm researching and not just blindly copying and pasting.
  • I'm okay with asking my peers for help, it's a last resort and I only do it if I'm absolutely stuck and with people I trust.
  • I'm confident with asking coaches for help, if they're easy to reach and talk to I will ask for help before I get too stuck, I still like to do research by myself first.
  • I'm confident with reflection, it's very useful as it helps me see what I did good and what didn't go so well. It helps me improve and be more efficient for next time.

Reluctance To Ask For Help

I can't think of a specific scenario but it's definitely something that has happened to me before. The main reasons I've been reluctant are because I was scared of what people would think of me especially if I feel like the task is "easy", and that I was eventually able to complete the task by myself. Next time, I have to do a better job of convincing myself that it's okay to ask for help no matter what and that most questions aren't stupid.
